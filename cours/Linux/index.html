Linux

Definition

Linux est un code opensource, un noyau, il a pour but d'etre utiliser comme un programme utlisiable par la carte mère. Il va traduire du code en instruction informatique.
Linux est composer de distributions, ce sont des système d'exploitation qui utilise le noyau Linux (Ubuntu, Debian, ...). Une distribution peut etre basé sur une autre distribution, on les appel des "fork" ( "fourchette", des copies).

Virtualisation

La virtualisation est de transferer un os sur une machine (l'emulation est de faire du code pour transferer une machine a travers du code). Ceci en sort en deux grand concept :

  • Paravirtualisation: VM pensé vitualisation, Kernel modifier, Drivers opti. C'est une virtualisation qui utlise un système pour exister.

  • Virtualisation hybride: Tous les OS compatible, instruction CPU, Meilleur perf. C'est une virtualisation qui existe a coter des autre virtualisation

Terminal vs Shell

  • Terminal: application qui permet d'afficher lignes des commandes (vs code)

  • Shell: langage qui permet de donner des instruction a travers de ligne de commandes (bash, batch, powershell). Ils sont souvent pensé pour un seul systeme d'exploitation

un Prompt: user@machine:~path
informations données avant la commande
user: lutilisateur auquel vous êtes connecté a l'instant
machine: Le nom de la machine auquel on est connecter a l'instant. tips: il peut s'agir d'une ip
path: il s'agit de l'endroit ou on est situé dans les fichier de la machine
il est interessant de comprendre un prompt car les shell ne sont pas toujours les même sur un serveur (voir batch, powershell,...).

Droits et permission

Linux est composer de plusieur utilisateur, chaque processus tournent et appartient a 1 seul groupe et 1 utilisateur. il existe 3 type d'ulisateur :

  • untilisateur reel: a un nom d'utilisateur, a un UID (indentifiant systeme), d'autre propriété personnelle, des permissions.

  • utilisateur systeme: uniquement pour un processus, plage d'UID (géner par le systeme), ne peut pas etre utiliser par l'humain, des permissions. (revoir les notions d'utilisateur)

  • root: est l'administrateur: est un utilisateur reel, il a absoluement tout les droits sula machine, a un UID est egale à 0, linux ne fourni pas toujours le mot de passe, il est très dangereux de se connecter au root, est seulement recommander pour la création de l'utilisateur et de groupe.

groupe: il a un nom unique, a un GID (indentifiant systeme), des utilisateur reels ou non, des permissions
chaque processus doit avoir un seul et meme groupe.
tout utilisateur, lors de sa création, lui créer un groupe par defaut avec comme GID l'ID de l'utilisateur

Commande de base

Une commande est un texte que l'on écrit dans le terminal afin d'executé une action. Une application est obligatoirement des commandes

Pour exectuer en tant qu'administrateur il faut utiliser la commande sudo (voir a quoi correspond le sudo) qui est native a Ubuntu.

  • apt permet d'installer des applications via un shell
    Apt (Advenced packaging tool) est une application native de linux qui foncionne par paquet (donc un gestionnaire de paquet) une sorte de store. La difference avec un store est qu'il n'y a pas d'entreprise derriere.

  • ls [option] permet d'afficher le contenu du dossier ou on se situe

    • -l affiche plus de données
    • -lh affiche en octet
    • *.zip affiche les fichiers contenant .zip

  • cd [direction] permet de se deplacer dans les dossier

  • pwd permet d'afficher l'url ou l'on se situe

  • cp [option] [source] [destination] permet de copier une sources

  • mv [option] [source] [destination] permet de renomer un fichier et le deplacer

  • mkdir [nom] permet de créer un dossier

  • touch [nom] permet de créer un fichier

  • rm [option] [nom] permet de supprimer

commande droit et permission

  • su [user] permet de changer d'utilisateur

  • whoami permet d'afficher l'utilisateur

  • groups [user] permet d'afficher les groupes de l'utilisateur

  • adduser [user] permet de créer un utilisateur (n'est pas native a linux est originaire de debian), demandera ensuite un mdp et qlq informations perso (optionelle)
    Il va créer un un dossier utlisateur dans home /home/user
    permet egalement d'assigner un utilisateur a un groupe adduser [user] [group]

  • deluser [user] permet de supprimer un utilisateur
    Il ne va pas supprimer les dossier utlisateur dans home /home/user
    permet egalement de retirer un utilisateur d'un groupe deluser [user] [group]

  • passwd permet de changer le mdp de l'utilisateur en devant renseigner le mdp précedent.
    Cependant si l'utilisateur est root, ayant tous les droit il forcera le changement sans renseigner le mdp précedent, si l'utilisateur est un autre utilisateur il n'aura pas la permission.

  • addgroup [group] permet de créer un groupe

  • delgroup [group] permet de supprimer un groupe

  • chown [user]:[group] [fichier/dossier] permet d'attribuer les droits de propriété sur un fichier/dossier a un utilisateur et/ou a un groupe
    Si l'on veut changer uniquement les droit pour un utilisateur, il faut faire chown [user], mais si l'on veut changer les droits pour un groupe il faut faire chown :[group]

  • chmod [option] [recherche] permet de gerer les droit d'acces d'un fichier/dossier:

    • qui changer? u (utilisateur), g (groupe), o (autre)

    • quoi faire? + (ajout), - (retrait)

    • quoi changer? r (Read), w (Write), x (eXectue)

    Rappel: pour voir les droits faire ls -l

combo de commande

il est possible d'ecire plusieur commande a la suite.
cmd a | cmd b : exectuera la cmd a qui sera donner a cmd b
cmd a > cmd b : exectuera la cmd a et ecrasera le contenu du fichier cmd b
cmd a >> cmd b : exectuera la cmd a et mettra a la suite dans le contenu du fichier cmd b
chaque commande une fois ecxecute sort un type d'information :

  • stdin lecture qui va demander de remplir qlq chose a l'utilisateur

  • stdout affiche dans la console une réponse de type standard

  • stderr affiche dans la console une reponse de type erreur

Communication

Une ip est une adresse sur internet, il en exite 2 version ip: ipv4 et ipv6
Lors de de communication entre machine il y a 2 ip qui utiliser :

  • privée: elles servent sur les réseaux locaux

  • publique: elles servent a intéragir avec internet

Un protocole est une maniere d'envoyer un type donnée :

  • http/https (HyperText Transfer Protocol): protocole utiliser pour transmettre une page web, HTTPS est chiffré avec le protocole SSL

  • ftp/sftp (File Transfert Protocol): protocole utiliser pour transmettre des fichier, SFTP est chiffré grace au secure Shell

  • ssh (Secure SHell): protocole utiliser pour se connecter au terminal d'une autre machine

[user]@[hostname] : [user] -> utilisateur de la machine du serveur
[hostname] -> IP du serveur auquel on souhait se connecter
Pour utiliser le protocole SSH il est nécessaire qu'il soit installer apt-get install openssh-server
